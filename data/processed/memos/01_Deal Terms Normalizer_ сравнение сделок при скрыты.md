# Deal Terms Normalizer: сравнение сделок при скрытых upfront

## Суть (1-2 предложения)
Строим сервис, который превращает текст пресс-релиза о сделке в «каноническую схему условий» и позволяет искать/сравнивать похожие сделки по структуре, даже когда суммы (upfront и др.) не раскрыты. Для аналитиков life sciences (BD, competitive intelligence, инвест-аналитика), особенно в R&A.

## Проблема
В пресс-релизах часто скрыты ключевые финансовые параметры (upfront, детализация milestone’ов), поэтому сравнение сделок «по размеру» ломается. При высокой частоте сделок аналитикам всё равно нужно быстро понимать типовую структуру и паттерны (опции, этапность, royalties, milestones) и находить аналоги.

## Решение
LLM-пайплайн, который:
- извлекает термины сделки из текста релиза,
- нормализует формулировки в единый словарь/таксономию,
- сохраняет сделку как структурированную карточку,
- делает similarity search «похожие по структуре сделки» и UI для сравнения бок-о-бок.

## MVP (3-6 месяцев)
1) Публичная JSON-схема Deal Terms Schema (тип сделки, актив/модальность, стадия, geography, эксклюзивность, опции, milestones (тип/триггеры), royalties (наличие/диапазон если есть), co-dev/co-com, лицензия/покупка и т.п.).  
2) Экстракция из пресс-релиза (LLM + правила) с оценкой уверенности по полям.  
3) Нормализация терминов (словарь синонимов, маппинг «option to license» и т.д.).  
4) Векторизация карточек + similarity search.  
5) Мини-каталог (например, 300–1000 релизов) и UI сравнения 2–3 сделок.

## Почему сейчас
- Рост активности сделок «по всей индустрии», включая R&A, повышает нагрузку на аналитиков и ценность автоматизации.  
- LLM-экстракция и эмбеддинги сделали возможным устойчивое структурирование и «поиск по смыслу/структуре» без ручной разметки.

## Голубой океан
Фокус не на оценке стоимости, а на структурном сравнении там, где финансы скрыты — это закрывает «слепую зону» традиционных data-провайдеров. Ниша защищается комбинацией: (а) канонической схемы, (б) накопленного корпуса нормализованных карточек, (в) workflow «загрузил релиз → получил аналоги».

## Путь к traction
- Wedge: бесплатный инструмент «Similar Deals from Press Release» (вставил текст → топ-10 похожих).  
- Community hook: открытая Deal Terms Schema + open-source мини-парсер.  
- Дистрибуция: LinkedIn/Slack-комьюнити BD/biopharma, рассылка «Deal structure of the week», интеграции через API в внутренние пайплайны.

## Монетизация
B2B подписка: расширенный каталог, пакетная обработка, экспорт, командный доступ; API для similarity/нормализации (usage-based). Платные кастомные схемы/таксономии для корпоративных knowledge-base.

## Exit thesis
Покупатели через 2–3 года: life sciences data/analytics платформы, инструменты для BD/competitive intelligence, корпоративные knowledge-base/enterprise search. Мотив: ускорить deal intelligence и добавить дифференциатор «структурные аналоги при скрытых суммах».

## Риски и митигация
1) Низкая точность извлечения из кратких релизов → confidence scoring, human-in-the-loop для платных клиентов, активное обучение на ошибках.  
2) Неоднозначность терминов → строгая схема, словарь синонимов, хранение evidence spans (цитаты из текста) для аудита.  
3) Недостаточная масса карточек для similarity → начать с узкого сегмента (R&A), open-source для сбора корпуса, партнёрства с нишевыми медиа/агрегаторами.

## Доказательства
- Ограничение: «A lot of the financial details were masked in that.»; «It's unclear what the upfront was, for example, and so forth.»  
- Тренд: «it's good to see this deal making happening across the industry.»; «I'm always happy when it's happening in the R &A world.»
